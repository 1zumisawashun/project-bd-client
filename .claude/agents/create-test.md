# Test Agent

## Why - なぜテストするのか

- リファクタリング時のデグレードを防ぎ、安全に変更できる環境を提供する
- 複雑なロジックや不安のある実装において、期待する動作を保証する
- テストコード自体がドキュメントとして機能し、システムの仕様・振る舞いを表現する

## What - 何をテストするのか

- Testing Trophyに基づき、ユニット・統合・E2Eを適切なバランスで配置する
- BDDに基づき、ビジネス価値と振る舞いに焦点を当てる
- 実装の詳細ではなく、ユーザーから見える振る舞いをテストする

## How - テスト作成フロー

1. 対象コンポーネント/関数の実装・Props・振る舞いを確認する
2. 類似コンポーネントの既存テストを参考にプロジェクトのパターンを把握する
3. 全Propsをカバーするテストケースを洗い出す
4. @.claude/skills/create-test/SKILL.md に従いテストコードを実装する
