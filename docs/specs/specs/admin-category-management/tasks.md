# Implementation Plan

## Tasks

- [ ] 1. データアクセス層の実装
- [x] 1.1 (P) カテゴリー取得関数の作成
  - データベースから全カテゴリーを取得する関数を実装する
  - 作成日時の降順でソートする
  - 既存の DB クエリパターンに従う
  - _Requirements: 2.1_

- [x] 1.2 (P) カテゴリー作成の Server Action 実装
  - 管理者権限の確認ロジックを実装する
  - カテゴリー名の重複チェックを行う
  - 新規カテゴリーをデータベースに保存する
  - ActionsResult 形式でレスポンスを返却する
  - _Requirements: 1.1, 1.4, 1.5_

- [ ] 2. バリデーションスキーマの定義
- [x] 2.1 カテゴリーフォームのバリデーションスキーマを作成
  - Zod を使用してカテゴリー名のバリデーションを定義する
  - 必須チェック（空文字不可）を実装する
  - 最大文字数（100文字）の制限を設定する
  - 日本語のエラーメッセージを設定する
  - _Requirements: 1.2, 1.3_

- [ ] 3. UIコンポーネントの実装
- [x] 3.1 (P) カテゴリー一覧コンポーネントの作成
  - カテゴリー名と作成日を表示するリストを実装する
  - カテゴリーが存在しない場合の空状態メッセージを表示する
  - 既存のコンポーネントパターンに従う
  - _Requirements: 2.1, 2.2, 2.3_

- [x] 3.2 カテゴリー作成フォームコンポーネントの作成
  - カテゴリー名入力フィールドを持つフォームを実装する
  - react-hook-form + zodResolver でバリデーションを統合する
  - 送信中のローディング状態を表示し、送信ボタンを無効化する
  - 成功・エラーメッセージを表示する
  - Server Action との連携を実装する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 4.1, 4.2, 4.3, 4.4_

- [ ] 4. 管理者カテゴリーページの実装
- [x] 4.1 管理者向けカテゴリー管理ページの作成
  - ページレベルでの管理者認可チェックを実装する
  - 未認証ユーザーはログインページへリダイレクトする
  - 非管理者ユーザーは適切なページへリダイレクトする
  - カテゴリー一覧の初期データを取得して表示する
  - カテゴリー作成フォームを統合する
  - _Requirements: 2.1, 3.1, 3.2, 3.3_

- [ ] 5. 統合と動作確認
- [x] 5.1 フォーム送信後のカテゴリー一覧更新
  - カテゴリー作成成功後に一覧を再取得・更新する
  - ページのリフレッシュまたは revalidation を実装する
  - _Requirements: 1.1, 2.1_

- [x]\* 5.2 E2E動作確認テスト
  - 管理者としてカテゴリーを作成できることを確認する
  - バリデーションエラーが正しく表示されることを確認する
  - 重複カテゴリー名でエラーが表示されることを確認する
  - 非管理者がアクセスするとリダイレクトされることを確認する
  - _Requirements: 1.1, 1.3, 1.4, 3.1, 3.2_
